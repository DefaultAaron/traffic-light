# 标注工具使用指南

本项目提供两个标注工具，用于第二阶段方向标注的审阅与编辑。两个工具界面与操作方式一致，仅数据源不同。

| 脚本 | 数据集 | 图片数 | 标注格式 |
|------|--------|--------|----------|
| `scripts/annotate_s2tld.py` | S2TLD 原始子集 | ~1,222 | Pascal VOC XML（已有） |
| `scripts/annotate_bstld.py` | BSTLD 测试集 | 8,334 | YAML → Pascal VOC XML（首次运行自动生成） |

---

## 一、环境准备

工具依赖 `Pillow`（图像处理）和 `PyYAML`（BSTLD 脚本），均已在项目 `pyproject.toml` 中声明。

```bash
# 确认依赖已安装
uv sync
```

---

## 二、启动

```bash
# S2TLD 标注工具
uv run python scripts/annotate_s2tld.py

# BSTLD 标注工具
uv run python scripts/annotate_bstld.py
```

**BSTLD 首次启动**会自动读取 `data/raw/BSTLD/test/test.yaml`，为每张图片生成对应的 Pascal VOC XML 文件至 `data/raw/BSTLD/annotations_fix/`。此过程为幂等操作——已存在的 XML 不会被覆盖，后续启动会跳过。

## 五、标注操作

### 5.1 修改类别

点击标注列表中的**类别名称**（如 `red`），弹出类别选择菜单，选择新类别即可完成修改。

可用类别（第二阶段 7 类 + 特殊类）：

| 类别 | 含义 |
|------|------|
| `red` | 红灯（圆形） |
| `yellow` | 黄灯（圆形） |
| `green` | 绿灯（圆形） |
| `redLeft` | 红色左转箭头 |
| `greenLeft` | 绿色左转箭头 |
| `redRight` | 红色右转箭头 |
| `greenRight` | 绿色右转箭头 |
| `off` | 灭灯 |
| `wait_on` | 等待亮灯（仅 S2TLD） |

### 5.2 添加标注

1. 在图片区域**鼠标左键拖拽**绘制矩形框（青色虚线预览）
2. 松开鼠标后，选框保持显示（青色实线），同时弹出类别选择菜单
3. 选择类别后，标注自动添加至列表并显示在图片上，选框消失
4. 选择 `Cancel` 或点击菜单外区域取消，选框自动清除

### 5.3 修改框坐标

点击标注行中的**坐标文字**（如 `(741,273)→(751,295)`），坐标会变为 4 个输入框（xmin、ymin、xmax、ymax），可直接编辑数值。

- 按 `Enter` 确认修改
- 按 `Escape` 取消修改
- 点击 **✓** 确认，**✗** 取消
- 验证规则：必须为整数，且 xmin < xmax、ymin < ymax

### 5.4 删除标注

点击标注行右侧的 **✕** 按钮即可删除该条标注。

### 5.5 控制单条标注显隐

- 取消勾选标注行左侧的**复选框**，可隐藏该条标注的框线（不影响保存）
- 重新勾选恢复显示

### 5.6 切换全部标注显隐

- 点击顶栏 **[A] Hide All** 按钮或按 `A` 键：隐藏全部标注
- 再次点击或按 `A` 键：显示全部标注
- 隐藏全部后，仍可单独勾选某条标注使其显示

## 六、保存机制

- **自动保存**：任何修改（修改类别、添加、删除）后 500ms 无操作即自动保存至 XML 文件
- **导航保存**：切换图片（上/下一张）前会立即保存所有未写入的更改
- **退出保存**：通过 `Ctrl+Q` 或点击窗口关闭按钮退出时，均会在退出前完成保存
- **无修改不写入**：仅浏览不编辑时，不会对磁盘文件产生任何写入操作
- 保存成功时，顶栏右侧会短暂显示绿色 `Saved` 提示

## 七、数据路径

### S2TLD

| 内容 | 路径 |
|------|------|
| 图片 | `data/raw/S2TLD/JPEGImages/*.jpg` |
| 标注（XML） | `data/raw/S2TLD/Annotations-fix/*.xml` |

### BSTLD

| 内容 | 路径 |
|------|------|
| 图片 | `data/raw/BSTLD/test/rgb/test/*.png` |
| 原始标注（YAML） | `data/raw/BSTLD/test/test.yaml` |
| 编辑标注（XML） | `data/raw/BSTLD/annotations_fix/*.xml` |

BSTLD 标注工具首次运行时从 YAML 生成 XML，后续所有编辑均基于 XML 文件。原始 YAML 不会被修改。

---

## 八、标注格式说明

两个工具均使用 Pascal VOC XML 格式存储标注。每张图片对应一个 XML 文件，结构如下：

```xml
<annotation>
    <filename>24068.png</filename>
    <size>
        <width>1280</width>
        <height>720</height>
        <depth>3</depth>
    </size>
    <object>
        <name>red</name>
        <pose>Unspecified</pose>
        <truncated>0</truncated>
        <difficult>0</difficult>
        <bndbox>
            <xmin>741</xmin>
            <ymin>273</ymin>
            <xmax>751</xmax>
            <ymax>295</ymax>
        </bndbox>
    </object>
</annotation>
```

编辑时工具会完整保留 `pose`、`truncated`、`difficult` 等字段的原始值，不会因编辑而丢失这些信息。

---

## 九、BSTLD 类别映射

BSTLD 原始标注使用大写英文类别名，首次生成 XML 时会自动映射为统一命名：

| BSTLD 原始类别 | 映射后类别 |
|---------------|-----------|
| `Red` | `red` |
| `Yellow` | `yellow` |
| `Green` | `green` |
| `off` | `off` |

BSTLD 测试集仅包含以上 4 类，方向标注（`redLeft`、`greenLeft` 等）需通过工具手动添加。

## 十、常见问题

**Q: 启动时提示 "No XML files found"**
A: S2TLD 工具需要 `data/raw/S2TLD/Annotations-fix/` 目录下存在 XML 文件。BSTLD 工具会自动从 YAML 生成 XML，但需确保 `data/raw/BSTLD/test/test.yaml` 存在。

**Q: 图片太小，看不清交通灯细节**
A: 使用滚轮（或触控板双指滑动）或 `+`/`-` 键缩放，缩放以鼠标位置为中心。缩放后可用鼠标右键拖拽（触控板双指点按后拖拽）或中键拖拽进行平移。按 `Ctrl+0` 恢复原始大小。

**Q: 误操作删除了标注怎么办？**
A: 当前版本不支持撤销。如果尚未保存（500ms 内），可立即切换到其他图片再切回来重新加载。如果已保存，需手动编辑 XML 文件恢复。

**Q: 标注框线遮挡了图片内容**
A: 按 `A` 键隐藏全部标注框线，或在右侧面板取消勾选特定标注的复选框。
