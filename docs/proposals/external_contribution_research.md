# 交通灯检测领域前沿研究与项目贡献机会报告 (2026-04-27)

> **状态**：研究报告。
> **目的**：综合 2025–2026 年交通灯检测（TLD）领域最新文献，评估现有开发计划，并识别本项目可向外部贡献的高价值空间。

---

## 1. 2025–2026 领域前沿进展

### 1.1 核心架构演进
- **YOLOv12 & 高级变体**：YOLO 系列依然是部署侧基准，但在 2025 年通过引入 R-ELAN 和 Area-based Attention 进一步提升了 mAP（~96%）。
- **SPD-Convolution (Space-to-Depth)**：替代了传统下采样，在处理远距小目标时通过保留像素细节显著减少信息损失。
- **Vision Transformers (ViT) 集成**：DINOv2 等预训练 backbone 被整合入 DETR 类架构，显著提升了在极低分辨率下的特征提取能力。
- **Open-Vocabulary 检测**：如 YOLOE 等模型通过文本/图像提示实现对未知交通灯配置的零样本检测，填补了长尾配置空白。

### 1.2 物理层与工程化痛点
- **Vortex-Induced Vibration (风致抖动)**：研究表明高架龙门架在风载下的微小抖动是导致感知端 bbox 振荡/闪烁的主因，非 ML 算法本身缺陷。
- **Road Arrow Integration (车道指引)**：利用路面标记判定交通灯相关性已取代 HD-Map 成为 SOTA 的鲁棒 fallback 方案。
- **Fourier Domain Adaptation (FDA)**：在不增加额外标注的前提下，通过对齐频谱特性，实现跨环境（如晴天到雾天）的有效域适应。

---

## 2. 现有计划评估与增强建议

基于最新研究，对现有计划提出以下增强方向：

### 2.1 架构层增强 (整合入 `development_plan.md`)
- **注入 SPD-Conv**：建议在 YOLO26 / YOLOv13 的 backbone 中引入 SPD-Conv，以提升远距小目标检测能力，且不会带来额外的推理延迟。
- **引入 FDA**：在 R2 训练 Pipeline 中增加 FDA 模块，提升模型在未标注恶劣天气数据上的鲁棒性。

### 2.2 推理与集成层增强 (整合入 `temporal_optimization_plan.md`)
- **高通滤波消除抖动**：针对龙门架抖动问题，在 tracker 层加入针对 <5Hz 频率的 bbox 震动抑制逻辑。
- **SAHI + 地图门控**：利用 OSM 地图信息作为门控信号，仅在接近路口时触发高开销的 SAHI/切片推理，确保 Orin 计算预算（50ms）内的最大精度。

### 2.3 R2 数据采集 SOP (必备决策)
- **增加 "Road Arrow" 类别**：将路面指引箭头纳入联合检测清单，作为判定信号灯关联性的关键特征。

## 3. 外部贡献空间识别

本项目的 R2 产出是该领域极度稀缺的资源。

### 3.1 极高优先级贡献机会 (R2 主线副产品)
1. **中国域 + 现代龙门 Gantry 数据集**：业内公开集以欧美垂直装灯为主，本项目采集的现代中国域龙门架数据可弥补这一严重地域空白。
2. **TL × Barrier 联合检测基准**：业内首个结合交通信号与道闸管理的联合数据集，具备高度工程应用价值。
3. **标准化困难场景切片**：利用已识别的 demo8-15 痛点（逆光/黄昏/强光斑），发布标准化的困难切片协议，而非笼统发布全集。

### 3.2 高价值工程实践
1. **TRT-OSS 部署指南**：针对 DEIM/Transformer 类架构在边缘侧部署面临的 Deformable Attention 算子缺失问题，发布构建指南。
2. **ByteTrack + EMA 双语言对齐基准**：公开已有的 fixture 测试集，提供跨语言（Python/C++）工程落地时的性能对齐参考。
